# VibeFeed (kreds)

VibeFeed er en lille dansk social app til lukkede vennekredse: et kronologisk feed (X-agtigt layout: avatar-kolonne, header-linje med @handle · tid, medie i afrundet kort) med opslag, billeder, korte videoer (maks. 6 sekunder, maks. 25 MB — optages med kameraet i appen eller vælges fra biblioteket; afspilles lydløst i loop i feedet) og meningsmålinger, indlejrede kommentartråde (sammenklappet som standard — boble/“Vis kommentarer (X)” folder tråd + kommentarfelt ud), en like-økonomi (du kan modtage ét like mere, end du selv har givet), private kredse, vennesøgning, notifikationer og profiler (med bio). Ét tryk på et billede eller en video åbner en fuldskærms-lightbox (`js/lightbox.js`): billeder kan pinch-zoomes (1x-4x), panoreres og dobbelttryk-zoomes (1x ↔ 2.5x) via Pointer Events (side-zoom er slået fra i viewporten); videoer vises med lyd, native controls og loop. Dobbelttryk på mediet i feedet liker stadig opslaget (320 ms tap-vindue) uden at åbne lightboxen. Feedbaren har en lup ved venstre kant, der folder en inline kreds-søgning ud og filtrerer kreds-pillerne live ("Hele kredsen" vises altid). Alle rækker i Søg (venner og globale resultater) åbner profilpanelet; profiler for ikke-venner viser en rød "Tilføj til din kreds"-chip (add_friend) i stedet for "I din kreds siden …", og deres opslag er skjult af RLS ("Bliv venner for at se opslag"). Ven-profiler har omvendt en lille grå "Fjern ven"-knap under relations-linjen (skjult på egen profil og for `@vibefeed`-botten): bekræftelses-popup → `remove_friend`-RPC, der sletter venskabet BEGGE veje — profilen forbliver synlig (panelet gen-renderes til ikke-ven-tilstanden med Tilføj-chippen), og fælles kreds-medlemskaber påvirkes ikke. Opslag til hele kredsen kan deles via systemets dele-ark (navigator.share, ellers kopieres teksten); opslag i private kredse kan ikke deles. Egne opslag kan redigeres og slettes; andres opslag kan anmeldes via ⋯-menuen (ved 2 anmeldelser skjuler serveren opslaget for alle). Den officielle profil `@vibefeed` er automatisk ven med alle, og dens opslag vises fastgjort øverst i "Hele kredsen". Notifikationer kan tappes for at hoppe direkte til opslaget (eller åbne profilen). Kredse styres via medlems-sheetet (tap på medlems-headeren i en kreds): fjern medlemmer (ved 3+ medlemmer opretter serveren en Ja/Nej-afstemning i kredsen), invitér venner (invitationer altid — `add_kreds_member` sender en invitation, som modtageren accepterer/afviser i Notifikationer; ved 3+ medlemmer udløser accept en afstemning i kredsen) og forlad kreds; "Ny kreds" inviterer de valgte venner (kun opretteren er medlem fra start), og "Ny kreds"-sheetet har en "Vælg alle"/"Fravælg alle"-knap over vennelisten. Topbaren skjules ved scroll ned (feedbar/søgefelt glider op til toppen) og vises igen ved scroll op eller nær toppen. En rød prik på hjerte-fanen viser nye realtime-hændelser (likes/kommentarer/invitationer/anmodninger/nye opslag — kun session, ryddes når fanen åbnes). Kreds-piller i feedbaren får desuden en lille rød prik, når kredsen har opslag, brugeren ikke har set endnu: per enhed gemmes hvornår hver kreds sidst blev åbnet (localStorage `vf_feed_seen`), og både boot/refetch (én posts-forespørgsel) og realtime-INSERTs tænder prikken; den slukkes når kredsen åbnes. Nye/usete opslag i selve timelinen markeres med et lille rødt "NY"/"NEW"-mærke højrestillet i header-linjen (før ⋯-knappen): per enhed gemmes ISO-tidspunktet for det nyeste opslag, brugeren allerede har haft på skærmen (localStorage `vf_post_seen`) — flag beregnes ved hver `loadPosts`-render mod det gemte tidsstempel, og tidsstemplet rykkes først frem EFTER render og kun når feedet faktisk ses (hjem-fanen aktiv + `document.hasFocus()`), så mærkerne overlever indtil brugeren reelt kigger; egne opslag, det fastgjorte `@vibefeed`-opslag og teasers er undtaget. Notifikationslisten fremhæver tilsvarende ulæste rækker (rød 7px-prik før avataren + chip-tonet baggrund, klassen `.unread`) for alt med `created_at` nyere end sidste åbning af akt-fanen (localStorage `vf_notif_seen` — gemmes EFTER rendering, når fanen er den aktive visning, så fremhævningen ses under selve besøget og er ryddet ved næste); invitations-/anmodningsrækker deltager også, og hjerte-fanens prik (`#tabdot`) er uændret. Profiler har en samtykke-styret aktivitetsvisning: "Del min aktivitet"-toggle i Rediger profil (`profiles.show_activity`) og en "Se aktivitet"-chip på ven-profiler (`activity_allowed`/`activity_of`-RPC'er — slår du selv fra, kan du heller ikke se andres). Appen understøtter dansk og engelsk (`js/i18n.js`): første start uden gemt sprog viser en fuldskærms sprogvælger ("Dansk"/"English" — uden flag) FØR login/boot, og sproget kan altid skiftes i Rediger profil ("Sprog / Language"-chips — skifter statisk markup via `data-i18n`/`data-i18n-ph`/`data-i18n-aria`-attributter OG gen-renderer den synlige dynamiske UI med det samme). Valget er per enhed (localStorage `vf_lang`, `<html lang>` følger med); serverskabt feed-indhold (governance-afstemninger som "Afstemning: …", " — Vedtaget ✅/Afvist ❌" og det fastgjorte velkomstopslag) forbliver dansk by design. Slet konto-bekræftelsesordet er SLET på begge sprog. I iOS-appen sender `auth.js` device-token + bruger-id + sprog (`lang`) + reklame-samtykke (`consent`) til den native notifikations-bro (`issue_device_token`/`revoke_device_token`, `window.webkit.messageHandlers.vibefeed`) — der postes friske creds ved sprogskifte, så native notifikationer følger sproget; ren no-op i browsere. Backend er Supabase (auth, Postgres, storage, realtime). Appen er en ren statisk side uden build-step og er også pakket som iOS-app (WKWebView i `ios/`).

Reklamer & privatliv: appen viser (kommende) reklamer via Appodeal/Google AdMob. Første start uden gemt valg (localStorage `vf_consent`) viser en fuldskærms samtykke-skærm (`#consentview` — efter sprogvalget, kan ikke lukkes uden valg): "OK — også personlige reklamer" (`personal`) eller "Kun ikke-personlige reklamer" (`limited`). Valget kan altid ændres under Rediger profil → Privatliv (to chips + link til politikken), og hver ændring postes til den native bro som `{type:"consent", value}` (`setConsent`/`getConsent` i `js/helpers.js`). Privatlivspolitikken er to selvstændige statiske sider uden scripts: `privatliv.html` (dansk) og `privacy.html` (engelsk) — links i appen (samtykke-skærmen, Rediger profil og linjen under opret-knappen på signup) følger sproget via `policyURL()`/`data-policy-link` i `js/i18n.js`. `FORTEGNELSE.md` er den interne GDPR artikel 30-fortegnelse (ikke linket fra appen).

Live: deployes automatisk via Vercel — hvert `git push` til `main` udløser et deploy.

## Filoversigt

| Fil | Indhold |
| --- | --- |
| `index.html` | Kun markup: head (CSP m.m.), hele appens HTML samt script-tags nederst. Statiske tekster har `data-i18n`/`data-i18n-ph`/`data-i18n-aria`/`data-i18n-title`/`data-i18n-alt` (dansk fallback i markup) |
| `css/app.css` | Hele stylesheetet (flad Instagram-agtig stil) |
| `js/vendor/supabase.js` | Vendored `@supabase/supabase-js` 2.110.0 (UMD-build, urørt) |
| `js/config.js` | Supabase-URL/nøgle, `sb`-klienten, recovery-detektion af hash |
| `js/i18n.js` | Sprog-laget: komplette DA/EN-ordbøger, `t(key, vars)`, `getLang`/`setLang` (localStorage `vf_lang`), `initI18n(onChange)`, `applyStaticI18n()` (data-i18n-attributter), pluralisering (`likesLabel`/`stemmerLabel`) og `dateLocale()` |
| `js/store.js` | Delt mutérbar tilstand: `me` (+`setMe`), `USERS`, `state`, composers, `pv`, `curTab` m.m. |
| `js/helpers.js` | Småhjælpere: `esc`, `el`, tidsformat (`fmtTime` — sprogafhængig via i18n), avatarer/gradienter, `toast`, `imgUrl`, `uuid`, `BADGE()`, reklame-samtykke (`getConsent`/`setConsent`/`pushConsentToBridge` — localStorage `vf_consent` + native bro) |
| `js/auth.js` | Login, opret, glemt/nulstil adgangskode, `boot()`/`resetApp()`, auth-events, native notifikations-bro (`pushNativeCreds`/`nativeLogout` — kun aktiv i WKWebView) |
| `js/feed.js` | Hent/render feed (X-layout, `postHTML`, fastgjort `@vibefeed`-opslag), feedbar med kreds-søgning (lup → inline filter af pillerne), slørede teasers fra private kredse (`kreds_teasers`-RPC, `teaserHTML`, anmod-om-at-være-med), likes + saldo (qchip), deling (`sharePost`), faner (`switchTab`), timeline-klik (enkelt-tryk på medie åbner lightboxen, dobbelttryk liker — 320 ms vindue), rediger/slet egne opslag, anmeld andres opslag (`reports`), skjul-topbar-ved-scroll (`resetBarHide` + `.hidebar` på body), ulæste kreds-opslag (`vf_feed_seen`, `refreshUnseen`/`markFeedSeen`/`markFeedUnseenRT` — rød `.pdot` på kreds-pillerne), NY-mærker på nye opslag i timelinen (`vf_post_seen` — flag i `loadPosts`, `.newchip` i `postHTML`) |
| `js/comments.js` | Kommentartråde (sammenklappet som standard, `cmtSectionHTML`/`toggleCmtSection`), composer, kommentar-likes, billede i kommentar |
| `js/kredse.js` | "Ny kreds"-sheet (`create_feed`-RPC — inviterer de valgte venner) og medlems-sheet (`remove_kreds_member`/`add_kreds_member` (= send invitation)/`leave_kreds`-RPC'er, ejer-tag, forlad kreds) |
| `js/compose.js` | Skriv-skærmen: tekst, medie-menu (kamera/video/bibliotek), billede- og videovedhæftning (video maks. 6 s / 25 MB), meningsmåling-editor, tegn-ring, del-til-valg |
| `js/polls.js` | Meningsmålinger: view-model (`mapPoll`), rendering (`pollHTML`) og stemmeafgivning (`votePoll`) |
| `js/search.js` | Søg-fanen: venner + global søgning + `add_friend`; hele rækken åbner profilpanelet |
| `js/lightbox.js` | Fuldskærms-lightbox: billede med pinch-zoom/pan/dobbelttryk-zoom (Pointer Events, uafhængigt af side-zoom), video med lyd + native controls; scroll-lås og safe-area-luk-knap |
| `js/notifications.js` | Aktivitets-fanen (likes, svar, nye venner, kreds-anmodninger med Godkend/Afvis for kreds-ejeren, kreds-invitationer med Accepter/Afvis via `accept_kreds_invite`/`decline_kreds_invite`); ulæst-fremhævning af rækker siden sidste åbning (`vf_notif_seen` — `.unread`/`.udot`); notifikations-prikken på hjerte-fanen (`setNotifDot`/`realtimeNotify`); tap på en række hopper til opslaget (med flash-highlight) eller åbner profilen |
| `js/profile.js` | Egen profil (statistik, bio, rediger-sheet med "Del min aktivitet"-toggle og "Sprog / Language"-chips, avatar, slet konto-popup, log ud), venne- OG ikke-ven-profil (bio, vennetal via RPC'er, "Tilføj til din kreds"-chip for ikke-venner, "Fjern ven"-knap med bekræftelses-popup (`remove_friend`-RPC) for venner, "Se aktivitet"-chip + aktivitets-sheet), bobler |
| `js/realtime.js` | Realtime-kanal og debounced refetch ved ændringer/fokus; INSERT-lyttere på `kreds_invites`/`kreds_requests` føder notifikations-prikken |
| `js/main.js` | Importerer alt, kobler alle event-lyttere (`init*()`), initialiserer i18n (sprogskifte-callback der gen-renderer synlig UI), viser første-start-sprogvælgeren (`#langview`) og samtykke-skærmen (`#consentview` — når `vf_consent` mangler) og kører opstartssekvensen |
| `privatliv.html` | Privatlivspolitik (dansk) — selvstændig statisk side, inline styling, ingen scripts |
| `privacy.html` | Privacy policy (engelsk) — selvstændig statisk side, inline styling, ingen scripts |
| `FORTEGNELSE.md` | Intern GDPR artikel 30-fortegnelse over behandlingsaktiviteter (ikke linket fra appen) |
| `kreds-app.html` | Gammel demo — rør den ikke |
| `ios/` | iOS-wrapper-projekt (holdes separat, rør den ikke herfra) |

## Kør lokalt

```
python3 -m http.server 8000
```

Åbn derefter `http://localhost:8000` i browseren. Ingen afhængigheder eller build — filerne serveres som de er.

## Deploy

Push til `main` — Vercel bygger og deployer automatisk.
